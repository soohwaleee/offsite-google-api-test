## 🔍 실행 중 발견·해소된 제약 (새 블로커 5건)

### N1. `Campaign.start_date` / `end_date` 필드 v23에서 **리네임**
- **변경**: `start_date` → `start_date_time` / `end_date` → `end_date_time`
- **포맷**: `"yyyy-MM-dd HH:mm:ss"` (기존 `YYYYMMDD`에서 변경)
- **end_date_time 제약**: 반드시 `23:59:59`로 끝나야 함 (`DATE_RANGE_ERROR_END_TIME_MUST_BE_THE_END_OF_A_DAY`)
- **영향**: PRD v3 및 체크리스트의 "start_date YYYYMMDD" 기술은 **outdated** — 업데이트 필요

### N2. `CampaignBudget.explicitly_shared` 기본값 문제
- **증상**: 누락 시 `CAMPAIGN_CANNOT_USE_SHARED_BUDGET`
- **해결**: `setExplicitlyShared(false)` 명시 설정 필수
- **교훈**: 새 버전에서 기본값 변경된 듯. 명시적 설정 권장

### N3. `Campaign.contains_eu_political_advertising` **v23 신규 필수 필드**
- **증상**: 누락 시 `REQUIRED` 에러, field_path: `contains_eu_political_advertising`
- **해결**: `setContainsEuPoliticalAdvertising(EuPoliticalAdvertisingStatus.DOES_NOT_CONTAIN_EU_POLITICAL_ADVERTISING)` 설정
- **중요**: v23에서 신규 도입된 필수 필드. EU DSA 규제 대응. **ad-center 운영 코드에도 반드시 적용 필요**

### N4. `AssetGroupListingGroupFilter.listing_source` 필수
- **증상**: 누락 시 `REQUIRED` 에러, field_path: `listing_source`
- **해결**: `setListingSource(ListingGroupFilterListingSource.SHOPPING)` 설정
- **체크리스트 업데이트 필요**: T1-4 스펙에 반영

### N5. Kotlin-Java 상호운용
- `CampaignOperation.Builder.setCreate()` 가 `Campaign`과 `Campaign.Builder` 둘 다 받아 overload ambiguity 발생
- 해결: `.apply { create = campaign }` 스타일 권장
