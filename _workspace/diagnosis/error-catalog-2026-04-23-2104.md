## 2. 주요 발견

### ① 같은 에러 묶음 현상
에러 하나를 유발해도 **관련 에러가 연쇄로 쏟아짐** (특히 Asset/AssetGroup 관련).
예: Case 5 는 12개 에러 동시 발생.
→ **첫 번째 에러만 사용자에게 표시**하는 게 UX 적으로 안전.

### ② 예산 0 vs 음수 = 같은 에러
Google 은 `< minimum` 으로 통일해서 처리.
→ 프론트엔드에서 `amount <= 0` 은 입력 단에서 막는 게 좋음 (API 호출하기 전에).

### ③ `CANNOT_SET_DATE_TO_PAST` + `START_DATE_AFTER_END_DATE` 동시 발생
종료일만 과거로 바꾼 케이스여도 "시작일 > 종료일" 에러도 따라옴.
→ 날짜 관련 에러는 **"종료일이 과거" vs "시작일 > 종료일"** 을 프론트에서 먼저 구분해서 메시지 뿌리기.

### ④ Google 중복 링크를 조용히 dedupe
중복 Asset 링크 자체는 에러 아님. Google 이 최종 유니크한 개수만 카운트.
→ 프론트에서 같은 Asset 을 같은 field_type 에 2번 못 붙이게 막아도 되고, 백엔드에서 dedupe 해도 안전.

### ⑤ 구조 에러가 값 에러보다 먼저
존재하지 않는 item_id 테스트했는데 **"필수 필드 누락"** 에러 발생.
→ Google 은 트리 구조 (parent-child) 를 먼저 검증, 값은 그 다음.
실제 "없는 item_id" 자체의 에러 코드는 올바른 구조에서 다시 테스트 필요.

## 3. 프로덕션 에러 매핑 테이블 (제안)

광고주 UI 에서 보여줄 때 Google 에러 코드를 아래 매핑으로 변환:

```kotlin
fun mapGoogleAdsError(codes: List<String>): UserFriendlyError = when {
    codes.any { it.contains("MONEY_AMOUNT_LESS_THAN") || it.contains("TOO_LOW") } ->
        UserFriendlyError("budget", "일 예산이 너무 낮습니다. 최소 ₩__ 이상 설정해주세요.")
    
    codes.any { it.contains("CANNOT_SET_DATE_TO_PAST") } ->
        UserFriendlyError("end_date", "종료일은 오늘 이후로 설정해주세요.")
    
    codes.any { it.contains("START_DATE_AFTER_END_DATE") } ->
        UserFriendlyError("end_date", "종료일은 시작일보다 나중이어야 합니다.")
    
    codes.any { it.contains("NOT_ENOUGH_HEADLINE_ASSET") } ->
        UserFriendlyError("headlines", "제목(Headline)을 최소 3개 이상 추가해주세요.")
    
    codes.any { it.contains("REQUIRED_BUSINESS_NAME_ASSET_NOT_LINKED") } ->
        UserFriendlyError("brand", "비즈니스명이 연결되지 않았습니다.")
    
    codes.any { it.contains("REQUIRED_LOGO_ASSET_NOT_LINKED") } ->
        UserFriendlyError("brand", "로고 이미지가 연결되지 않았습니다.")
    
    else ->
        UserFriendlyError("unknown", "광고 생성 중 오류가 발생했습니다. 관리자에 문의해주세요.")
}
```

### 우선순위 규칙
복수 에러가 동시에 오면:
1. 예산 관련 (`budget_error`) → 최우선
2. 날짜 관련 (`date_range_error`, `CANNOT_SET_DATE_TO_PAST`) → 두 번째
3. 자산 부족 (`NOT_ENOUGH_*`) → 세 번째
4. Brand Guidelines (`REQUIRED_BUSINESS_NAME`, `REQUIRED_LOGO`) → 네 번째
5. 기타 → "관리자 문의" 폴백
