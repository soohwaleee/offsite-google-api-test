## Key Findings

### 1. FieldMask 패턴 동작 확인 (v23)

3개 테스트 모두 동일한 패턴으로 성공:

```kotlin
val updated = Campaign.newBuilder()
    .setResourceName(existing)
    .setName(newName)           // 바꾸고 싶은 필드만 set
    .setEndDateTime(newEndAt)
    .build()

val op = CampaignOperation.newBuilder().apply {
    update = updated
    updateMask = FieldMasks.allSetFieldsOf(updated)   // set 된 필드만 자동 마스킹
}.build()
```

- `FieldMasks.allSetFieldsOf()` 헬퍼가 v23 에서도 정상 동작
- 임포트 경로: `com.google.ads.googleads.lib.utils.FieldMasks` (v23 네임스페이스 아님 — lib/utils)
- `resource_name` 은 업데이트 식별자로 사용되며 mask 에서 자동 제외됨

### 2. PMax 캠페인의 수정 가능 필드 확인

이번 라운드에서 검증된 것:
- ✅ `campaign.name` 수정 가능
- ✅ `campaign.end_date_time` 연장 가능 (검토 중 상태에서도)
- ✅ `campaign.status` PAUSED ↔ ENABLED 토글 가능 (검토 중 상태에서도)
- ✅ `campaign_budget.amount_micros` 수정 가능

### 3. 검토 중 상태에서도 status 전환 가능 (T6)

T6 에서 중요한 관찰:
- 생성 직후 AssetGroup 은 **"정책 검토 중"** 상태 (이전 세션 질의 참고)
- 그럼에도 불구하고 `campaign.status = ENABLED` 전환 성공
- Google Ads 는 검토를 **비동기로 병렬 처리** — status 전환과 정책 승인은 독립적
- **프로덕션 시사점**: 광고주가 "캠페인 켜기" 눌러도 즉시 노출되지 않고 정책 검토 완료 후 실제 송출

### 4. 예산 단위 재확인 (T5)

```
₩10,000 → amount_micros = 10_000_000_000  (10,000 * 1,000,000)
₩30,000 → amount_micros = 30_000_000_000
```

PRD v3 구현 시 입력단에서 KRW → micros 변환 필요.
