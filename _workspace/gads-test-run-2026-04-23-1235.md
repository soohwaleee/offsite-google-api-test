## Key Finding (v23 중요 변화점)

### Brand Guidelines 기본 활성화

PMax 캠페인은 v23 시점에 **Brand Guidelines 가 기본 ON** 상태로 생성됩니다.
이 모드에서는:

- `BUSINESS_NAME`, `LOGO` → **CampaignAsset 으로 링크 필수** (Campaign level)
- 나머지 모든 asset → **AssetGroupAsset 으로 링크** (AssetGroup level)

### 첫 시도 시 발생한 에러 (참고)

모든 자산을 AssetGroupAsset 으로만 링크하려 하면 3가지 에러가 뜹니다:

```
campaign_error: REQUIRED_BUSINESS_NAME_ASSET_NOT_LINKED
  "Brand Guidelines is enabled. Performance Max campaigns with Brand
   Guidelines enabled require at least one business name to be linked
   as a CampaignAsset."

campaign_error: REQUIRED_LOGO_ASSET_NOT_LINKED
  "... require at least one square logo to be linked as a CampaignAsset."

asset_link_error: BRAND_ASSETS_NOT_LINKED_AT_CAMPAIGN_LEVEL
  "Brand Guidelines is enabled. Performance Max campaigns with Brand
   Guidelines enabled must link business name and logo assets as
   CampaignAssets."
```

### 대안

Brand Guidelines 를 끄고 싶으면 캠페인 생성 시 `brand_guidelines_enabled = false` 설정 가능.
다만 **기본값이 ON** 이므로 프로덕션에서도 ON 기준으로 설계하는 게 자연스러움 (이번 테스트는 그대로 진행).

## Asset 생성 방식

### Text Asset
```kotlin
Asset.newBuilder()
    .setType(AssetType.TEXT)
    .setTextAsset(TextAsset.newBuilder().setText("무신사 특가"))
    .build()
```

### Image Asset
```kotlin
Asset.newBuilder()
    .setType(AssetType.IMAGE)
    .setName("test-marketing-xxx")
    .setImageAsset(
        ImageAsset.newBuilder()
            .setData(ByteString.copyFrom(pngBytes))
    )
    .build()
```

이미지는 `BufferedImage` + `ImageIO.write("PNG")` 로 메모리에서 생성 (테스트용 플레이스홀더).
프로덕션에서는 무신사 공식 자산(로고, 마케팅 이미지) 으로 교체.
